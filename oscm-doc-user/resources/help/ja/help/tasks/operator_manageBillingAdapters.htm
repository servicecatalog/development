Managing billing systems

The platform supports the integration of third-party or proprietary billing systems. Technology providers have the choice of using the platform's billing system or an external billing system of their choice when integrating their applications with the platform.

On the Manage billing systems page, you can link an external system to the platform. For successfully linking a system, the following prerequisites must be fulfilled:

  • The connection properties specified by the operator need to point to the external billing system.

One of the supported billing systems must be set as default. The default billing system is predefined for technology providers who register an application as a technical service to make it available on the platform.

When accessing the Manage billing systems page, the details of the billing system that is currently configured as default are expanded. On the page, you can:

  • View the list of supported billing systems.
  • Add a new billing system.
  • Update the name and connection properties of a billing system.
  • Set the default billing system.
  • Remove a billing system.

To define a new billing system:

  1. Click Add. This inserts an empty section for defining the billing system details.
  2. Enter the Billing System ID of the billing system. This ID uniquely identifies the billing system in the platform. Once the new billing system is defined, the ID cannot be changed.

    IDs are restricted to 40 characters and must not contain any of the following characters:

    ! " # $ % & ' * + , / : ; < = > ? \ ^ `

  3. Enter the Name of the billing system.

    Names are restricted to 40 characters. Any character is allowed.

  4. Enter the Connection Properties. Depending on the billing adapter implemented to connect the platform with the external billing system, the number of connection properties and their semantic can be different. However, they must point at the specific external billing system and specify such data as the port with which the billing system is connecting, or a provider URL. The connection between the billing system and the adapter is done using JNDI lookup.

    Enter the name of a property in the first field and the corresponding property value in the second field.

    Use the plus sign to add an additional property, the minus sign to remove a property.

  5. Click Test Connection to make sure that the connection to the billing system can be established. A corresponding message is shown telling you whether the connection was successfully established.
  6. Click Set as Default to define the new billing system as default billing system, if required. The new billing system is instantly marked as default. The marker is removed from the previous default system.
  7. To define the billing system, click Save.

Technology providers can now select this billing system when registering an application as a technical service on the platform.

To update an existing billing system:

  1. Expand the details of the billing system to be updated.
  2. Change the Name of the billing system, if required.
  3. Change the values of the connection properties. Use the plus sign to add an additional property, or the minus sign to remove a property, if required.
  4. Click Test Connection to make sure that the connection to the billing system can still be established.
  5. Click Set as Default to define the billing system as default.
  6. Click Save for your changes to become effective.

To remove a billing system:

  1. Expand the details of the billing system to be deleted.

    The following prerequisites must be fulfilled for deleting a billing system:

    • The billing system must not be marked as default.
    • There is no technical service, marketable service and no active subscription that uses the billing system.
  2. Click Delete and confirm the deletion.

    The billing system is instantly deleted.
